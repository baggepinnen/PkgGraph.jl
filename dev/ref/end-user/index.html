End-user

PkgGraph.PkgGraph — Module

Visualize the dependency graph of a Julia package.

Use PkgGraph.open to view the graph in the browser, or PkgGraph.create to generate an image locally.

(Note that these functions are not exported).

See PkgGraph.Internals for more functions.

source

Online

PkgGraph.open — Function
open(pkgname; kw...)

Open the browser to an image of pkgname's dependency graph.

The given package must be installed in the currently active project.

See Settings for possible keyword arguments.

source

Local

PkgGraph.create — Function
create(pkgname, dir = tempdir(); fmt = :png, open = true, kw...)

Render the dependency graph of the given package as an image in dir, and open it with your default image viewer. Uses the external program 'dot' (see graphviz.org), which must be available on PATH.

The given package must be installed in the currently active project.

fmt is an output file format supported by dot, such as :svg or :png.
If fmt is :svg, the generated SVG file is post-processed, to add light and dark-mode CSS.

To only create the image, without automatically opening it, pass open = false.

See Settings for more keyword arguments.

source

Settings

jll

Whether to include binary 'JLL' dependencies in the graph (default: true)

stdlib

Whether to include packages in the standard library (default: true)

style

Custom Graphviz styling. Default: default_style.

base_url

See url. By default, the first entry in webapps.
